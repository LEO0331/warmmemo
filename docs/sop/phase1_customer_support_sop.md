# WarmMemo Phase 1 SOP（客服版）

適用範圍：Spark 方案、無 webhook 自動對帳、付款與加值含人工核對流程。  
版本：v1.0

## 1) 目標與原則

- 目標：在 1 個工作天內完成用戶問題分流、回覆與必要交接。
- 原則：先確認身份，再確認訂單/點數，再給下一步可執行指引。
- 禁止事項：客服不得直接承諾「已付款即時生效」，需以「人工核對後更新」為準。

## 2) 客服日常檢查（每日 2 次）

- 時段建議：10:30、16:30。
- 檢查項目：
  - 新增 `topupRequests`（status=`pending`）
  - 訂單卡在 `checkout_created` 超過 24 小時
  - 用戶回報「已付款但未更新」

## 3) 工單分流標準

- A 類（付款核對）：用戶表示已付款，但狀態未變更。
- B 類（點數不足）：用戶無法使用進階功能，需加值。
- C 類（帳號/權限）：無法登入、角色/頁面權限異常。
- D 類（內容使用）：草稿生成、匯出、通知顯示問題。

## 4) 標準處理流程

### A 類：已付款未更新

1. 請用戶提供：`付款單號(invoiceId)`、付款時間、註冊 Email。  
2. 客服在後台定位訂單（方案名稱 + userId + 建立時間）。  
3. 交由管理員核對後更新 `paymentStatus`。  
4. 回覆用戶完成通知。

回覆模板：
> 我們已收到你的付款回報，正在進行人工核對。  
> 核對完成後會更新你的方案狀態，通常於 1 個工作天內完成。  
> 若你方便，也可提供付款單號以加速處理。

### B 類：點數不足 / 加值申請

1. 指引用戶在彈窗提交「加值申請」。  
2. 確認是否已產生 `topupRequests/pending`。  
3. 交由管理員審核加點。  
4. 完成後回覆新餘額。

回覆模板：
> 目前你的點數已用完，我們已收到加值申請。  
> 審核完成後會由系統更新餘額並通知你，感謝耐心等待。

### C 類：登入/權限

1. 確認用戶是 Email/Password 登入。  
2. 確認是否為同一帳號（Email 與 UID 一致）。  
3. 若為管理員頁面權限，交由管理員核對 `users/{uid}.role` 與 `admins/{uid}`。

### D 類：功能使用（草稿/匯出）

1. 先確認點數是否足夠。  
2. 若功能報錯，請用戶提供畫面與錯誤碼。  
3. 建立技術工單並附：操作步驟、帳號 UID、發生時間。

## 5) SLA 建議

- 首次回覆：4 小時內（工作時段）。
- 付款核對完成：1 工作天內。
- 加值審核完成：1 工作天內。
- 高優先（無法登入/全站性）：即時通知管理員。

## 6) 升級與交接規則

- 需交接管理員的情境：
  - 付款狀態變更
  - 點數加點審核
  - 權限異常（admin/user）
- 需交接技術的情境：
  - 規則權限錯誤（permission-denied）
  - 查詢索引錯誤（failed-precondition）
  - 重複發生且可重現的功能錯誤

## 7) 客服紀錄欄位（每案必填）

- 使用者 Email / UID
- 問題類型（A/B/C/D）
- 訂單 ID（若有）
- 付款單號（若有）
- 處理動作
- 下一步與預估完成時間

## 8) 結案模板

> 你的案件已完成處理：  
> - 訂單狀態：{{status}}  
> - 付款狀態：{{paymentStatus}}  
> - 點數餘額：{{tokenBalance}}  
> 若還需要我們協助，直接回覆此訊息即可。
